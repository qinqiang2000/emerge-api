## 开场（立论）

过去做软件：画 UI、写按钮、让人去适应软件。现在反过来——给数字同事一个**目标**，它自己去做。emerge 是我们拿「文档处理」这件事做的一次完整实验。

## ① Software 3.0（框架）

1.0 写代码 → 2.0 训练权重 → **3.0 自然语言就是程序**。prompt / skill 就是新代码。
> 「以前改软件要改代码；现在改软件，改一句话。」

## ② 人格化：给目标，不点击　〔★必演〕

数字同事 = 你把意图说清楚，它补齐上下文、自己干。不是多一个表单字段、多一个对话框。

**〔演示〕** Cowork 里打一句：`找到北方工业项目，增加 gemini-2.5-flash，用它跑 1.jpg`。看它自己：列项目 → 加模型 → 列文档 → 建实验 → 跑提取，5 步无人工。
> 「我没点任何按钮，只说了一句话。」

## ③ 换前端 = 换 agent 客户端　〔★必演，最强〕

同一个 emerge，能力在后端；前端是 CLI、Cowork、Desktop、网页——随便换，能力不变。UI 退化成口味问题。

**〔演示·王牌〕** 一句：`对比 gemini-2.5-flash 和 gemini-3-flash-preview 在 1.jpg 上谁更准`。看它**自主编排**：建两实验 → 各跑提取（中途自己重试了一次 503）→ 各跑 eval → 出对比表（90.9% vs 81.8%）→ **自己诊断** merchantName 丢分原因 + 建议改 prompt。
> 「这一长串多步推理 + 自我诊断，没有一个是我写死的流程。」

## ④ 能力靠「包」，不是硬代码　〔★必演〕

新能力 = 装一个能力包/插件包（skill + 连接器 + /命令），装一次，所有客户端点亮。不是发版、不是改前端。

**〔演示〕** CLI：`/plugin marketplace add qinqiang2000/emerge-api` → `/plugin install emerge@emerge`；或 Cowork：Upload 插件 zip。装完立刻多出 `/emerge:run` `/emerge:compare`。
> 「给同事开一个新本事，像装个 App，不用我们重新写软件。」

## ⑤ 文件系统当「总线」，MCP 暴露文件操作　〔技术听众的 aha〕

传统做法：为每个能力写一套专用 API（list_models、get_prediction…几十个）。我们的做法：**核心对象本来就是文件**（project.json / models / prompts / predictions），所以只暴露 `ls / read / grep` 三个通用文件操作，一次性把所有对象的读取打通。

**〔演示〕** 让 agent 先 `ws_list("models/")`、`ws_read("项目/project.json")`——它「看文件」发现一切，不是「调专用接口」。
> 「不给它一百个专用接口，给它一根文件系统总线——它像同事一样自己翻文件。」

## ⑥ 扩展性：开大也不做多个应用　〔收尾〕

提取已跑通 ✓；接下来的**文档分类、文档匹配**，长在同一根总线、同一套「跑→纠正→评分→发布」闭环、同一个连接器上——不是再做三个应用，是同一个数字同事多一项本事。
> 「Documents in. APIs emerge. —— 喂进文档，涌现出 API，而且越纠正越准。」

## 演示编排

| 优先级 | 演示 | 证明 |
|---|---|---|
| 必演 1 | 一句话「加模型 + 跑文件」（②） | 给目标不点击 |
| 必演 2（王牌） | 一句话「对比两模型」→ 自动编排 + 出表 + 自诊断（③） | 换客户端 / agent 自主 |
| 必演 3 | 装插件 → `/emerge:run` 冒出来（④） | 能力包扩展 |
| 加演 | `ws_list / ws_read` 翻文件发现项目（⑤） | 文件系统总线（给工程师） |
| 对照 | 同开 emerge 网页版（有 review UI），说「这套 UI 不是必须的」（③） | 换前端=换客户端 |

**一条龙脚本（8–10 分钟）**：② 暖场 → ③ 王牌（全自动对比 + 诊断）→ 中间穿插 ⑤ 翻文件当「技术揭秘」→ ④ 装插件证明可复制 → ⑥ 分类/匹配收尾画饼。

**现场风险**：需联网 + Cowork 已连 emerge（OAuth 登录态在）。模型偶发 503 会自动重试——反而是好素材：「它自己处理了错误」。
