# 手工 Dogfood：修正驱动的 prompt 自优化闭环

对应功能 commit `cd8cef4`（feat(tune-loop)）。验证「在 review 改字段 → 系统记录 → 攒够后 agent 提议 tune → /improve → eval-gate 候选 → 一键采纳铸新 prompt 版本 → 可回滚」整条链。

## 前置

- dev servers 起着：backend `:8080`、frontend `:5173`（`./dev.sh`）。
- 一个有 **≥5 篇 docs + 已抽取 draft 预测** 的项目（本指引用 `振兴_testset`）。
- **extract / proposer 模型可用**。
  ```bash
  curl -s -XPUT localhost:8080/lab/projects/振兴_testset/models/active \
    -H 'Content-Type: application/json' -d '{"model_id":"m_default"}'
  ```
- 下文 `WS=backend/workspace/振兴_testset`，磁盘核查命令在 `backend/` 下跑。

---

## ① 修正落盘 + 计数器

1. 浏览器开项目 → 点 docs 树里任一文档进 review（URL 变 `?review=<文件名>`）。
2. 在 **✏ 校订稿** tab（唯一可编辑），点字段值（contentEditable）改几个，比如 invoiceType、currency、billToName（改 3~5 个，便于看计数跳）。
3. 点右上 **保存**。

**核查**（磁盘）：
```bash
python3 - <<'PY'
import json,glob
d=json.load(open(glob.glob('workspace/振兴_testset/reviewed/<文件名前缀>*.json')[0]))
print('keys:', list(d.keys()))
print('_corrections:', json.dumps(d.get('_corrections'),ensure_ascii=False,indent=2))
print('counter:', json.load(open('workspace/振兴_testset/project.json')).get('corrections_since_tune'))
PY
```
**预期**：`_corrections` 含你改的字段（每个 `{before, after}`，`before` 保留原类型）；`corrections_since_tune` = 本次修正字段数。
- 反向验证：再保存一次但不改任何字段 → 计数**不变**（仅非空修正才 +）。

---

## ② agent 主动提议 tune

前提：`corrections_since_tune ≥ 5` 且 reviewed 文档数 ≥ 5。若不足 5 篇 reviewed，可快速把若干 draft 确认为 reviewed：
```bash
python3 - <<'PY'
import json,glob,os,urllib.request,urllib.parse
slug='振兴_testset'
for d in sorted(glob.glob('workspace/%s/predictions/_draft/*.json'%slug))[:5]:
    fn=os.path.basename(d)[:-5]; ents=json.load(open(d)).get('entities',[])
    body=json.dumps({'entities':ents,'source':'manual'}).encode()
    u='http://localhost:8080/lab/projects/%s/reviewed/%s'%(urllib.parse.quote(slug),urllib.parse.quote(fn))
    urllib.request.urlopen(urllib.request.Request(u,data=body,headers={'Content-Type':'application/json'},method='POST')); print('OK',fn)
PY
```
1. 在 review 内打开 chat（右上对话图标），随便发一句中性的，如「我刚改完这篇的几个字段」。

**预期**：agent 回复开头出现 **「你已修正 N 处跨 M 篇文档，要我 /improve 一下 prompt 吗？」**（来自 `emerge_extractor.md` 的 nudge；只提议，不自动跑）。

---

## ③ /improve → 实时候选卡

1. chat 里发 `/improve`。agent 调 `start_job`，chat 出现 **进度卡**。

**预期**：
- 卡片状态 `running`，随后每轮（每轮真实抽取全部 reviewed，约 2~3 分钟/轮）刷新一行：
  `turn N · best acc 0.XX (turn K) · baseline 0.XX (Δ ±0.XX)`（这是 `formatJobLine` 增强后的格式）。
- 回退轮（分数没超过 best）**不被采纳**（job log 里 `saved:false`，带 `rationale`）。
- 有 **暂停 / 取消** 控件。

**核查**（实时）：`tail -f workspace/振兴_testset/jobs/j_*.jsonl`，看 `turn` 事件的 `field_accuracy_macro` 与 `saved`。

> 若这批 ground truth 已接近满分（baseline 很高）或修正是任意值，autoresearch 多半找不到改进 → 卡片显示 **"baseline still best"、无 Accept 按钮**（这是**正确门控**，不是 bug）。要走通 ④ 的真实采纳，需要 baseline 有真实可修复的差距（见下方"强制构造可采纳候选"）。

---

## ④ 采纳 → 铸新 prompt 版本

当某轮 best 真正超过 baseline，卡片出现绿色 Δ 块（`candidate +X acc vs baseline`）+ **Accept** 按钮。

1. 记下当前 `active_prompt_id` 和 `global_notes` 开头、prompt 数量：
   ```bash
   python3 -c "import json;d=json.load(open('workspace/振兴_testset/project.json'));p=json.load(open('workspace/振兴_testset/prompts/%s.json'%d['active_prompt_id']));print('active',d['active_prompt_id'],'| counter',d.get('corrections_since_tune'),'| gn[:40]',repr((p.get('global_notes')or'')[:40]))"
   ls workspace/振兴_testset/prompts/*.json | wc -l
   ```
2. 点 **Accept**。

**预期**（卡片 + 磁盘）：
- 卡片转为内联确认条：新变体 id + Δ，提示去 Prompts tab。
- 新 `prompts/pr_*.json`：label = **`tune <日期> (+X% acc)`**、`derived_from` = 原 active、**`global_notes` 与原 active 完全一致（未被清空）**、schema 字段数不变（描述级改动）。
- `project.json`：`active_prompt_id` = 新变体、`corrections_since_tune` 归零、prompt 数 +1。

### 强制构造可采纳候选（当真实 job 出不了改进时）
合成一个 improving 候选，直接打真实 accept 端点（验证 Phase A 代码路径）：
```bash
python3 - <<'PY'
import json,copy,urllib.request,urllib.parse
from pathlib import Path
slug='振兴_testset'; ws=Path('workspace')/slug
pj=json.load(open(ws/'project.json')); pv=json.load(open(ws/'prompts'/f"{pj['active_prompt_id']}.json"))
job='j_dogfood00001'; cand=ws/'versions'/'_candidate'/job; cand.mkdir(parents=True,exist_ok=True)
imp=copy.deepcopy(pv['schema']); imp[0]['description']=(imp[0].get('description')or'')+' [dogfood]'
json.dump({"turn":0,"schema":pv['schema'],"rationale":"baseline","field_accuracy_macro":0.90,"macro_f1":0.90,"per_field":[],"predictions":{},"ts":"x","notes_hit":[],"notes_hit_filtered":[]},open(cand/'turn_0.json','w'),ensure_ascii=False)
json.dump({"turn":1,"parent_turn":0,"schema":imp,"rationale":"clarified desc","field_accuracy_macro":0.96,"macro_f1":0.96,"per_field":[],"predictions":{},"ts":"x","notes_hit":[],"notes_hit_filtered":[]},open(cand/'turn_1.json','w'),ensure_ascii=False)
u='http://localhost:8080/lab/projects/%s/schema/accept-candidate'%urllib.parse.quote(slug)
r=urllib.request.urlopen(urllib.request.Request(u,data=json.dumps({'job_id':job,'turn':1}).encode(),headers={'Content-Type':'application/json'},method='POST'))
print(json.dumps(json.load(r),ensure_ascii=False,indent=2))
PY
# 用完清理：rm -rf workspace/振兴_testset/versions/_candidate/j_dogfood00001
```
注：`job_id` 必须匹配 `^j_[a-z0-9]{12}$`。

## ⑤ Prompts tab + 回滚

1. 回项目页，展开 docs 树下的 **prompts/**。

**预期**：新变体 `tune <日期> (+X% acc)` 已列出且带 ⭐（active）。

2. 回滚：切回旧变体
   ```bash
   curl -s -XPOST localhost:8080/lab/projects/振兴_testset/prompts/pr_baseline/activate -d '{}' -H 'Content-Type: application/json'
   ```
   刷新页面 → ⭐ 移回 Baseline，tune 变体仍保留（可再切）。**版本可逆，无需 git。**

---

## 红线自检（应始终成立）
- AutoResearch 不自动 promote：候选只高亮，Accept 永远人点。
- proposer 输入不含文档原文 / bbox / counterexample。
- 采纳铸版本时 `global_notes` 不丢。
- 计数器只在「有修正的保存」时 +，在 Accept 时归零。

## 收尾
`振兴_testset` 是 testset，dogfood 产生的 reviewed 文件 / tune 变体可随时删（`workspace/` 已 gitignore，不进仓库）。
