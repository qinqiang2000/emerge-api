# Dogfood 指引 · 修正 → tune 闭环

面向自己上手走一遍「改字段 → 一键优化 → 接受新版本」的完整路径。
用 `振兴_testset`(4 篇文档)。

---

## 一、动手步骤

### 0. 起点
- 启动 dev,打开 `振兴_testset`,进 review 模式。
- 干净态下顶部没有横幅(没有待消化的修正)。

### 1. 改字段 + 保存
- 任选一篇,改 1~2 个字段的值(例如把 `currency` 改成正确币种)。
- 按 **⌘S** 保存 —— 不该弹浏览器「网页另存」,右下角出现 **「已保存」** toast。
- 再开一篇,把**同一个字段**也改一次(制造跨文档的同字段信号),可选再改个别的字段。

### 2. 看横幅(非对话入口)
保存后顶部出现一条 ochre 横幅:
- 只改了 1 个字段 → 「字段 X 已被你修正 N 次 · **优化该字段**」
- 改了多个 → 「已修正 X, Y(共 N 个字段)· **优化这些字段**」

> 单改一次的字段也会进去,不会被「热点字段」挤掉。

### 3. 展开看「改了什么」+ 跳转核对
- 点横幅左侧文字 → 展开,逐字段列出你的 **before → after**(原值删除线、新值正常色)。
- 点某条 before→after → **跳到对应文档**、自动选中该字段、滚动定位 PDF。
  跨文档也行(点哪条就开哪篇)。
- 被你改过的字段在表单里特殊标记:**ochre 底 + 「已修正」徽章**,hover 看 before→after。
  用来快速复核「我到底改了哪些、改对没」。

### 4. 一键优化
- 点横幅右侧按钮(优化该字段 / 这些字段)。
- 右侧弹出**一个干净的新对话**,顶部一张进度卡,右下角 toast「已开始优化」。
- 进度卡会依次显示(均为大白话,无 turn / acc 行话):
  - 「**正在测量当前版本的准确率(还没开始改)…**」(先量当前准确率)
  - 「**本轮在尝试:**…」(这一轮把哪条字段说明改成了什么、为什么)
  - 「每一轮都要把所有已校阅文档重抽一遍,约数分钟/轮」(慢是正常的,别当卡死)
  - 进度行用「**第 N 轮 · 最佳准确率 …(第 M 轮)· 基线 …(Δ …)**」,不再出现 `turn`/`acc`

### 5. 接受或放弃
- 跑完若候选**优于基线(Δ > 0)** → 卡片出现 **「采纳此优化」** 按钮(原文案是 `接受 turn N`,已去行话)。
- 点「采纳此优化」→ 生成一个新版本并设为当前版本;**不点就什么都不变**(永不自动应用)。采纳后顶部 ochre 横幅会立即消失(修正已折进新版本)。
- 不满意 → 点 ✕ 取消;你的修正不会被消耗,横幅还在,可以继续改完再优化。

**重点观察**:① 单字段也能优化(不卡旧的 ≥5 门槛);② 多字段全部进入优化;
③ 全程在右栏一张卡里看得见;④ 是否变更由你点「采纳此优化」决定;
⑤「提升字段」只列你聚焦的那几个字段(非聚焦字段的抖动不会被算成成果)。

---

## 二、「优化该字段 / improve」背后做了什么

两个按钮是同一台引擎,只是范围不同:
- **优化该字段 / 这些字段**(横幅):聚焦你刚改的那几个字段。
- **/improve**(对话里发起):面向全部字段的广撒网,门槛更高(需要够多已校阅文档)。

### 输入
- **当前生效的字段说明**:每个字段一段自然语言「该怎么抽这个字段」的描述,外加全局备注。
- **你校阅过的文档(ground truth)**:作为评分的标准答案。
- **你最近的修正(before → after)**:仅在聚焦优化时,把「你把这个字段从 X 改成了 Y」
  作为重点信号喂进去。只给字段名 + 你填的文本值。

### 处理
1. **量基线**:用当前说明把所有已校阅文档重新抽一遍,逐字段算准确率,得到一个总分。
   - 聚焦优化:总分**只看你选的那几个字段**(别的字段不拉高也不拉低)。
   - 广撒网:总分看全部字段。
2. **每轮改写**:一个 proposer 模型**只重写字段的「说明」**(描述/instruction),
   参考这几个字段错在哪、以及你的修正,把说明写得更准。
   - 它**只改说明**,不会增删/改名/改类型字段,不会编造示例答案。
3. **重抽 + 重评**:用改写后的说明再抽一遍全部已校阅文档,重新算总分。
   分更高才留作候选(评分是闸门)。
4. **循环**:直到达到轮数上限或连续几轮没进步就早停。

### 输出
- 一个(或几个)**候选版本**:只有字段说明变了的新 prompt。
- **永不自动应用**。你看到 Δ(比基线高多少)和本轮 rationale,自己决定是否「采纳此优化」。
- 采纳后才落成新的生效版本;原版本仍在,可回滚。

### 几条不会越界的红线(帮助理解边界)
- 只改「字段说明」这一段文字 —— 不塞示例输入输出、不动字段结构。
- 进模型的只有字段名 + 文本值 + 你的文字修正 —— **没有坐标/框选、没有整页文档原文**。
- 反例(你标过的错例)只用于回归测试,**不进**优化时的 prompt。
- 是否变更永远由你点「采纳此优化」,引擎不会自己改生效版本。

> 一句话:你改值 → 引擎据此把「字段说明」重写得更准 → 拿你的标准答案验证有没有变好
> → 变好了给你候选 → 你拍板是否启用。
