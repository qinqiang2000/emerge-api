# M4 — Polish (chat UX + dark mode + export + verify) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring chat surface up to claude.ai parity (right-aligned user bubble, left-aligned agent without bubble, grouped tool-call pills with running/done streaming states, full markdown rendering, explicit light/dark/system theme toggle), close the publish loop with an export bundle (`schema.json` + frozen `version.json` + `curl_example.sh` + `README.md` zip), upgrade tool-failure UX (red pill + error_code copy + chat-level retry), and end-to-end verify the M2A→M2C `_notes` → autoresearch hint plumbing actually delivers notes into the proposer prompt.

**Architecture:** Frontend chat shell rebuilt around three discrete render units — `UserBubble`, `AgentMessage` (markdown), `ToolCallGroup` (a stripe wrapping a list of `ToolCallPill`s) — assembled by a pure `groupChatEvents()` reducer that chunks consecutive `tool_call` events into one group. A new `useTheme` Zustand store toggles `data-theme={light|dark|system}` on `<html>`, persisted to `localStorage`. `tokens.css` switches from `prefers-color-scheme` to attribute selectors so explicit toggle wins. Markdown rendering uses `react-markdown` + `remark-gfm` scoped to `agent_text` only (user text stays plain to prevent injection). Per-tool short-hint extractor (`lib/toolHint.ts`) maps `tool_name → result → "8 fields" / "v1" / "macro_f1=0.97"` etc., used by `ToolCallPill` collapsed view. Tool-failure UX extends `ToolCallPill` (red border, error_code prominent, chat-level "重试上一条" button on `ChatPanel`). Backend gains a new `app/api/routes/export.py` mounting `GET /lab/projects/{pid}/export?version={n}` that streams a zip composed in-memory (no temp files); the README markdown is rendered from a small `app/exports/readme_template.py`. `_notes` hint loop verified by 1 new integration test that loads reviewed-with-notes from disk, runs `run_autoresearch_loop` for 1 turn against a stub provider, and asserts the recorded `user_content` substring contains the notes text.

**Tech Stack:** Frontend — `react-markdown`, `remark-gfm`, `@fontsource/poppins`, `@fontsource/lora`, `@fontsource/jetbrains-mono`, Zustand (existing), Radix Tooltip (existing). Backend — Python `zipfile.ZipFile` to BytesIO + FastAPI `StreamingResponse`. Reuses existing `safe_project_id`, `version_path`, `parse_version_id`, KeyStore. No new dependencies on the backend.

**Spec reference:** `docs/superpowers/specs/2026-05-08-agent-native-design.md` §8.1 (three-pane shell), §8.5 (tool-call rendering), §8.6 (visual identity / brand palette), §11 M4 deliverables. ROADMAP M4 entry covers same scope; this plan also folds open follow-ups: "Markdown not rendered in chat", "agent is thinking… subtle indicator", "Cross-store refresh on agent tool events" partial (we make ToolCallPill carry enough info to avoid a full refresh).

## Hard rules (red lines — never violate)

1. **No new color literals in components.** Every color comes through CSS-var tokens in `theme/tokens.css`. Any `bg-[#...]` or hex literal in JSX is a code-review block.
2. **Markdown rendering applies only to `agent_text`.** User input stays as plain text — no `**bold**`, no images, no links rendered, to prevent prompt-injection or phishing through user-controlled content.
3. **Theme toggle persists to `localStorage` ONLY.** No backend roundtrip; theme is a per-browser setting.
4. **Export bundle README NEVER includes a real API key.** Always renders the literal placeholder `<your saved key>`. CI test asserts no `ek_[A-Za-z0-9_-]{32}` pattern in any generated bundle.
5. **`/lab/projects/{pid}/export` requires `safe_project_id`** like every other lab route. The route does NOT require an API key (it's a lab endpoint), but it returns 404 if `active_version_id` is absent.
6. **Chat layout changes do not break `ApiKeyRevealModal`'s portal.** The modal is rendered at `<body>` via Radix Portal and must remain force-modal (Esc + outside-click disabled, only "我已保存" closes).
7. **Tool-call short hint extractor must NOT throw on unexpected result shapes.** Always returns `string | null`. A render path that crashes on malformed tool results is worse than no hint.
8. **`useTheme` writes `data-theme` ONLY on `<html>`.** Not on `<body>` or any container — Tailwind dark-variants and CSS-var fallback rely on the html element.

```typescript
// frontend/src/stores/theme.ts
import { create } from 'zustand'

export type ThemeMode = 'system' | 'light' | 'dark'

const STORAGE_KEY = 'emerge.theme'

function _isMode(s: string | null): s is ThemeMode {
  return s === 'system' || s === 'light' || s === 'dark'
}

function _writeAttribute(mode: ThemeMode): void {
  if (mode === 'system') {
    document.documentElement.removeAttribute('data-theme')
  } else {
    document.documentElement.setAttribute('data-theme', mode)
  }
}

interface State {
  mode: ThemeMode
  setMode: (mode: ThemeMode) => void
  apply: () => void          // re-apply current mode (no localStorage read)
  hydrate: () => void        // read localStorage, set mode, apply
}

export const useTheme = create<State>((set, get) => ({
  mode: 'system',
  setMode: (mode) => {
    set({ mode })
    _writeAttribute(mode)
    try { localStorage.setItem(STORAGE_KEY, mode) } catch { /* private mode etc. */ }
  },
  apply: () => _writeAttribute(get().mode),
  hydrate: () => {
    let stored: string | null = null
    try { stored = localStorage.getItem(STORAGE_KEY) } catch { /* ignore */ }
    const mode: ThemeMode = _isMode(stored) ? stored : 'system'
    set({ mode })
    _writeAttribute(mode)
  },
}))
```

```tsx
const NEXT: Record<ThemeMode, ThemeMode> = {

  light: 'dark',

      type="button"

      onClick={() => setMode(NEXT[mode])}

    </button>

}
```

```tsx
/**
 * Render agent_text as markdown. Configured to:
 *  - support GFM tables / strikethrough / task lists / autolink
 *  - block raw HTML (skipHtml + no rehype-raw)
 *  - block <img> nodes (we override the renderer to null)
 *  - force external links to open in a new tab with rel=noreferrer
 */
export default function AgentMessage({ text }: Props) {
  return (
    <div className="text-fg-primary leading-relaxed font-body max-w-none">
      <ReactMarkdown
        remarkPlugins={[remarkGfm]}
        skipHtml
        components={{
          // Block <img> entirely — agent should never embed images
          img: () => null,
          a: ({ href, children, ...rest }) => (
            <a
              {...rest}
              href={href}
              target="_blank"
              rel="noreferrer noopener"
              className="text-accent-primary underline"
            >
              {children}
            </a>
          ),
          code: ({ className, children, ...rest }) => {
            const isBlock = (rest as { node?: { position?: { start?: { column?: number } } } }).node?.position?.start?.column === 1
            return isBlock ? (
              <code className={`${className ?? ''} font-mono`}>{children}</code>
            ) : (
              <code className="font-mono bg-subtle text-fg-primary px-1 py-0.5 rounded text-[0.9em]">
                {children}
              </code>
            )
          },
          pre: ({ children }) => (
            <pre className="bg-subtle text-fg-primary p-3 rounded overflow-x-auto text-sm">
              {children}
            </pre>
          ),
          h1: ({ children }) => <h1 className="font-heading text-xl mt-3 mb-2">{children}</h1>,
```

```typescript
// frontend/src/lib/toolHint.ts
/**
 * Map a tool result to a single-line collapsed hint, e.g. "8 fields", "v1",
 * "macro_f1=0.97", "ek_abcdefgh". Returns null if no hint applies, the
 * tool is unknown, or the result shape is malformed.
 *
 * Discipline: NEVER throws. Render paths must be safe even when the agent
 * feeds back unexpected tool results.
 */
export function toolShortHint(toolName: string, result: unknown): string | null {
  if (result === null || result === undefined) return null
  const bare = toolName.replace(/^mcp__emerge_tools__/, '')

  // Helpers
  const tryParseJson = (s: string): unknown => {
    try { return JSON.parse(s) } catch { return undefined }
  }
  // Some legacy tool wrappers return a Python-repr string of a list/dict.
  const looksLikePyList = (s: string) => /^\[.*\]$/s.test(s.trim())
  const countPyDicts = (s: string): number | null => {
    // Count top-level dict braces — best-effort, not a full parser
    if (!looksLikePyList(s)) return null
    const m = s.match(/\{[^{}]*\}/g)
    return m ? m.length : 0
  }
  const asObj = (r: unknown): Record<string, unknown> | null => {
    if (typeof r === 'object' && r !== null) return r as Record<string, unknown>
    if (typeof r === 'string') {
      const parsed = tryParseJson(r)
      if (typeof parsed === 'object' && parsed !== null) return parsed as Record<string, unknown>
    }
    return null
  }

  switch (bare) {
    case 'derive_schema':
    case 'read_schema': {
      if (typeof result !== 'string') return null
      const n = countPyDicts(result)
      if (n === null) return null
      return n === 1 ? '1 field' : `${n} fields`
    }
    case 'list_docs': {
      if (typeof result !== 'string') return null
      const n = countPyDicts(result)
      return n === null ? null : `${n} docs`
    }
    case 'list_reviewed': {
      if (typeof result !== 'string') return null
      const n = countPyDicts(result)
      return n === null ? null : `${n} reviewed`
    }
    case 'list_projects': {
      if (typeof result !== 'string') return null
      const n = countPyDicts(result)
      return n === null ? null : `${n} projects`
    }
    case 'extract_batch': {
      const o = asObj(result)
      if (!o) return null
      const ok = typeof o.ok_count === 'number' ? o.ok_count : null
      const err = typeof o.err_count === 'number' ? o.err_count : null
      if (ok === null || err === null) return null
      return `${ok}/${ok + err} ok`
    }
    case 'score': {
      const o = asObj(result)
      if (!o || typeof o.macro_f1 !== 'number') return null
      return `macro_f1=${o.macro_f1.toFixed(2)}`
    }
    case 'freeze_version': {
      const o = asObj(result)
      if (!o || typeof o.version_id !== 'string') return null
      return o.version_id
    }
    case 'issue_api_key': {
      const o = asObj(result)
      if (!o || typeof o.key_prefix !== 'string') return null
      return o.key_prefix
    }
    case 'readiness_check': {
      const o = asObj(result)
      if (!o || !Array.isArray(o.checks)) return null
      const checks = o.checks as Array<{ status?: string }>
      const total = checks.length
      const pass = checks.filter(c => c.status === 'pass').length
      return `${pass}/${total} pass`
    }
    case 'contract_diff': {
      const o = asObj(result)
      if (!o) return null
      const added = Array.isArray(o.added) ? o.added.length : 0
      const removed = Array.isArray(o.removed) ? o.removed.length : 0
      const breaking = o.is_breaking === true ? ' (breaking)' : ''
      return `+${added} -${removed}${breaking}`
    }
    case 'start_job': {
      if (typeof result !== 'string' || !result.startsWith('j_')) return null
      return `job ${result.slice(0, 10)}`
    }
    case 'get_job': {
      const o = asObj(result)
      if (!o || typeof o.status !== 'string') return null
      return o.status
    }
    case 'pause_job':
    case 'resume_job':
    case 'cancel_job':
    case 'save_reviewed':
    case 'write_schema':
    case 'upload_doc':
      // Result is "ok" or doc_id — the pill name + checkmark is signal enough.
      return null
    default:
      return null
  }
}
```

```tsx
// frontend/src/components/Chat/ToolCallPill.tsx
import { useState } from 'react'
import { Check, X, Loader2, ChevronDown, ChevronRight } from 'lucide-react'

import type { ChatEvent } from '../../types/chat'
import { toolShortHint } from '../../lib/toolHint'


interface Props { event: Extract<ChatEvent, { type: 'tool_call' }> }


type PillState = 'running' | 'done' | 'error'


function pillState(e: Extract<ChatEvent, { type: 'tool_call' }>): PillState {
  if (e.ok === false) return 'error'
  if (e.tool_result === undefined || e.tool_result === null) return 'running'
  return 'done'
}


function extractErrorCode(result: unknown): string | null {
  if (typeof result === 'string') {
    try {
      const o = JSON.parse(result) as { error_code?: string }
      return o.error_code ?? null
    } catch { return null }
  }
  if (typeof result === 'object' && result !== null) {
    const o = result as { error_code?: string }
    return o.error_code ?? null
  }
  return null
}


export default function ToolCallPill({ event }: Props) {
  const [open, setOpen] = useState(false)
  const state = pillState(event)
  const displayName = event.tool_name.replace(/^mcp__emerge_tools__/, '')
  const hint = state === 'done' ? toolShortHint(event.tool_name, event.tool_result) : null
  const errorCode = state === 'error' ? extractErrorCode(event.tool_result) : null

  const Icon = state === 'running' ? Loader2 : state === 'error' ? X : Check
  const iconClass =
    state === 'running' ? 'text-fg-muted animate-spin' :
    state === 'error'   ? 'text-accent-danger' :
                          'text-accent-success'
  const Caret = open ? ChevronDown : ChevronRight

  return (
    <div
      data-state={state}
      className={
        'border-l-2 bg-surface text-sm transition-colors ' +
        (state === 'error' ? 'border-accent-danger' : 'border-accent-info')
      }
    >
      <button
        type="button"
        onClick={() => setOpen(o => !o)}
        className="w-full flex items-center gap-2 px-3 py-1.5 font-mono text-left hover:bg-subtle"
      >
```

```typescript
// frontend/src/lib/groupChatEvents.ts
import type { ChatEvent, RenderItem } from '../types/chat'


export function groupChatEvents(events: ChatEvent[]): RenderItem[] {
  const out: RenderItem[] = []
  let toolBuf: Extract<ChatEvent, { type: 'tool_call' }>[] = []

  const flushTools = () => {
    if (toolBuf.length > 0) {
      out.push({ kind: 'tools', calls: toolBuf })
      toolBuf = []
    }
  }

  for (const e of events) {
    if (e.type === 'tool_call') {
      toolBuf.push(e)
      continue
    }
    flushTools()
    if (e.type === 'user') out.push({ kind: 'user', text: e.text })
    else if (e.type === 'agent_text') out.push({ kind: 'agent', text: e.text })
    else if (e.type === 'error') out.push({
      kind: 'error',
      error_code: e.error_code,
      error_message_en: e.error_message_en,
    })
    // turn_end and any unknown type silently dropped
  }
  flushTools()
  return out
}
```

from __future__ import annotations

from typing import Any


_PLACEHOLDER_KEY = "<your saved key>"


def _field_row(f: dict[str, Any]) -> str:
    name = f.get("name", "")
    typ = f.get("type", "")
    desc = (f.get("description") or "").replace("|", "\\|").replace("\n", " ")
    enum = f.get("enum") or []
    if enum:
        desc = f"{desc} _(allowed: {', '.join(enum)})_"
    required = "yes" if f.get("required") else "no"
    return f"| `{name}` | {typ} | {required} | {desc} |"


def render_readme(*, project: dict[str, Any], version: dict[str, Any], project_id: str) -> str:
    """Render the README for an export bundle. Pure function — no I/O.

    The returned markdown explicitly uses `<your saved key>` as the API-key
    placeholder. NEVER inject a real key here; the caller has no business
    handing one to this function.
    """
    name = project.get("name", "(unnamed project)")
    version_id = version.get("version_id", "(unknown version)")
    model_id = version.get("model_id", "(unknown model)")
    fields = version.get("schema", []) or []
    notes = version.get("global_notes") or ""
    frozen_at = version.get("frozen_at", "")

    lines: list[str] = []
    lines.append(f"# {name} — extraction API ({version_id})")
    lines.append("")
    lines.append(f"- **Project ID:** `{project_id}`")
    lines.append(f"- **Active version:** `{version_id}` (frozen {frozen_at})")
    lines.append(f"- **Extraction model:** `{model_id}`")
    lines.append("")
    lines.append("## Field schema")
    lines.append("")
    if fields:
        lines.append("| Name | Type | Required | Description |")
        lines.append("|---|---|---|---|")
        for f in fields:
            lines.append(_field_row(f))
    else:
        lines.append("_(empty schema — this should not happen for a published version.)_")
    lines.append("")
    if notes.strip():
        lines.append("## Global notes")
        lines.append("")
        lines.append(notes.strip())
        lines.append("")
    lines.append("## Calling the API")
    lines.append("")
    lines.append("```sh")
    lines.append(f"curl -X POST https://<host>/v1/{project_id}/extract \\")
    lines.append(f'  -H "X-API-Key: {_PLACEHOLDER_KEY}" \\')
    lines.append(f"  -F file=@/path/to/document.pdf")
    lines.append("```")
    lines.append("")
    lines.append("## Response shape")
    lines.append("")
    lines.append("```json")
    lines.append('{')
    lines.append('  "entities": [')
    lines.append('    {')
    if fields:
        sample_lines = []
        for f in fields[:3]:
            n = f.get("name")
            t = f.get("type")
            if t == "number":
                sample_lines.append(f'      "{n}": 0')
            elif t == "boolean":
                sample_lines.append(f'      "{n}": true')
            elif t == "date":
                sample_lines.append(f'      "{n}": "2026-01-01"')
            else:
                sample_lines.append(f'      "{n}": "..."')
        lines.append(",\n".join(sample_lines))
    lines.append('    }')
    lines.append('  ],')
    lines.append('  "_evidence": [')
    lines.append('    { "field_name": 1 }')
    lines.append('  ]')
    lines.append('}')
    lines.append("```")
    lines.append("")
    return "\n".join(lines)

```python
# backend/app/exports/bundler.py
from __future__ import annotations

import io
import json
import zipfile
from pathlib import Path
from typing import Any

from app.exports.readme_template import render_readme
from app.workspace.paths import (
    project_json_path,
    schema_path,
    version_path,
)


_PLACEHOLDER_KEY = "<your saved key>"


class BundleVersionMissingError(Exception):
    """Raised when the requested version v{n}.json does not exist."""


def _curl_script(project_id: str) -> str:
    return f"""#!/usr/bin/env sh
# Edit HOST and PDF_PATH for your environment.
HOST="${{HOST:-https://example.com}}"
PID="{project_id}"
KEY="{_PLACEHOLDER_KEY}"
PDF_PATH="${{1:-/path/to/document.pdf}}"

curl -X POST "${{HOST}}/v1/${{PID}}/extract" \\
  -H "X-API-Key: ${{KEY}}" \\
  -F "file=@${{PDF_PATH}}"
"""


def build_zip_bundle(*, workspace: Path, project_id: str, version_n: int) -> bytes:
    """Compose the export zip for one project + version. In-memory only.

    Members:
      - schema.json     — the live lab schema (for reference; may differ from
                          frozen if the user has edited since publish)
      - version.json    — the immutable frozen blob (this is what /v1 serves)
      - curl_example.sh — working shell snippet with placeholder API key
      - README.md       — markdown intro with field table + curl + sample response
    """
    vp = version_path(workspace, project_id, version_n)
    if not vp.exists():
        raise BundleVersionMissingError(f"versions/v{version_n}.json missing")
    version_blob: dict[str, Any] = json.loads(vp.read_text())

    pj = project_json_path(workspace, project_id)
    project_blob: dict[str, Any] = json.loads(pj.read_text()) if pj.exists() else {}

    sp = schema_path(workspace, project_id)
    schema_blob: list[Any] = json.loads(sp.read_text()) if sp.exists() else []

    readme = render_readme(project=project_blob, version=version_blob, project_id=project_id)
    curl = _curl_script(project_id)

    buf = io.BytesIO()
    with zipfile.ZipFile(buf, mode="w", compression=zipfile.ZIP_DEFLATED) as z:
        z.writestr("schema.json", json.dumps(schema_blob, indent=2, ensure_ascii=False))
        z.writestr("version.json", json.dumps(version_blob, indent=2, ensure_ascii=False))
        z.writestr("curl_example.sh", curl)
        z.writestr("README.md", readme)
    return buf.getvalue()
```

```python
# backend/app/api/routes/export.py
from __future__ import annotations

import json
import re

from fastapi import APIRouter, HTTPException, Query
from fastapi.responses import StreamingResponse

from app.api.routes._safety import safe_project_id
from app.config import get_settings
from app.exports.bundler import BundleVersionMissingError, build_zip_bundle
from app.schemas.envelope import ErrorEnvelope
from app.workspace.paths import (
    parse_version_id,
    project_json_path,
)


router = APIRouter()


_FILENAME_SAFE = re.compile(r"[^A-Za-z0-9_.-]+")


def _envelope(code: str, msg: str) -> dict[str, str]:
    return ErrorEnvelope(error_code=code, error_message_en=msg).model_dump()


def _safe_filename(name: str, version_id: str) -> str:
    """Produce a filename safe for Content-Disposition. Drops anything that
    isn't alphanumeric / dot / dash / underscore."""
    base = _FILENAME_SAFE.sub("-", name).strip("-_") or "project"
    return f"{base}-{version_id}.zip"


@router.get("/lab/projects/{project_id}/export")
async def lab_export(
    project_id: str,
    version: int | None = Query(default=None, ge=1),
):
    safe_project_id(project_id)
    settings = get_settings()
    workspace = settings.workspace_root

    pj = project_json_path(workspace, project_id)
    if not pj.exists():
        raise HTTPException(status_code=404, detail=_envelope(
            "not_found", "project not found",
        ))
    project_blob = json.loads(pj.read_text())
    name = project_blob.get("name", "project")

    if version is None:
        avi = project_blob.get("active_version_id")
        if not avi:
            raise HTTPException(status_code=404, detail=_envelope(
                "not_published", "project has no active version; run /publish first",
            ))
        n = parse_version_id(avi)
        if n is None:
            raise HTTPException(status_code=500, detail=_envelope(
                "active_version_corrupt", f"active_version_id={avi!r} is not a valid version id",
            ))
    else:
        n = version

    try:
        blob = build_zip_bundle(workspace=workspace, project_id=project_id, version_n=n)
    except BundleVersionMissingError:
        raise HTTPException(status_code=404, detail=_envelope(
            "version_not_found", f"versions/v{n}.json does not exist",
        ))

    filename = _safe_filename(name, f"v{n}")
    return StreamingResponse(
        iter([blob]),
        media_type="application/zip",
        headers={"Content-Disposition": f'attachment; filename="{filename}"'},
    )
```

## Self-Review notes

- **Multi-entity `score()` / readiness / FieldEditor remains scope-cut.** Open follow-up #1 stays open after M4. Acceptable: M4 is polish, multi-entity is a quality fix that needs its own design pass (does FieldEditor support nested-table editing? what does `score()` return shape become?).
- **Per-tool retry button NOT implemented.** Chat-level "重试上一条" is the M4 minimum. Per-tool re-run requires a backend `/lab/chat/retry-tool {tool_use_id}` route that selectively re-invokes the SDK with the previous tool selected, plus chat-store wiring to splice the new result into the existing event log. Significant — M5 territory.
- **JobProgressCard left untouched in T10.** The card still uses its existing colors. If the new pill scheme makes the card feel out-of-place, a follow-up cosmetic PR can re-skin it; functional behavior is unchanged.
- **`_notes` integration test (T20) may surface that the loop runs the proposer LLM call AFTER scoring.** If it does, the test still passes because we capture all `provider.extract` calls and assert the substring across the union. If for some reason the proposer call is never made (e.g., the loop early-stops at turn 0), the test asserts `captured` is non-empty and will fail loudly.
- **Export bundle does NOT include `global_notes.md` as a separate file** — it's embedded into README. Rationale: keeps the bundle minimal; the consumer cares about behavior (README + curl), not lab editing artifacts.
- **`_safe_filename` regex strips Chinese characters.** A project named "中文项目" exports as `project-v1.zip` (the regex sub leaves only `-` then strip. Probably want to preserve the non-ASCII when generating the name. M5 polish.).
- **Tailwind `dark:` variants are now usable** thanks to T1's `darkMode: ['selector', '[data-theme="dark"]']`. Components that need *additional* dark-only styling (vs pure CSS-var swap) can now use `dark:bg-foo` / `dark:text-bar` classes. M4 components use only the CSS-var swap path; dark variants are a future polish lever.
- **No new MCP tools registered.** The publish toolset is unchanged from M3. The export endpoint is a `/lab/*` HTTP route, not an agent-callable tool — agents do NOT need to trigger exports.
- **Markdown rendering security stance.** `skipHtml` blocks raw HTML. `<img>` blocked at the renderer level (`img: () => null`). External links forced to `target=_blank rel=noreferrer noopener`. We do NOT run a full sanitizer like DOMPurify because react-markdown + skipHtml is sufficient for the threat model (agent is our own SDK, not arbitrary user input).
