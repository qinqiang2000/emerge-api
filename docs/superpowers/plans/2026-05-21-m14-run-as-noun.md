# M14 тАФ Run as first-class noun

> **Date:** 2026-05-21
> **Predecessor:** M12.x.d (in-session: `ScoreResultSummary` + `extract_batch` got `prompt_id`/`prompt_label`/`model_id`/`extract_model` echo fields; matrix `SummaryStrip` got the anchor chip. Down-payment commit.)
> **Trigger:** in one session three independent surfaces (chat narration, matrix grid header, review tabstrip) all failed to anchor "which (model, prompt) produced this". Three same-shape patches in one day = abstraction missing. The user named the pattern: *"цА╗цДЯшзЙф╕АчЫ┤хЬихРДчзНф┐о"*. Stepping back from the patch queue: emerge has no noun for "one extraction run." `predictions/_draft/{f}.json` is anonymous; `experiments/{id}/predictions/{f}.json` is identified only by filesystem location; `reviewed/_pending/{f}.json` carries `labeler_model` but nothing else. Every consumer reinvents the resolution.

## Goal

Promote **Run** to a first-class noun. Every prediction blob is self-describing тАФ it carries a `_run` envelope identifying (run_id, model, prompt, ts). Consumers (chat tool result, score summary, matrix UI, review tabstrip) read this single source. The three duplicated resolution paths collapse.

Concretely:
- `predictions/_draft/{f}.json` тЖР `extract_one` writes `_run` (active prompt ├Ч active model).
- `experiments/{exp_id}/predictions/{f}.json` тЖР `experiment_run` writes `_run` (experiment's prompt ├Ч experiment's model).
- `reviewed/_pending/{f}.json` тЖР `pre_label` writes `_run` (active prompt ├Ч labeler model).
- `score()` reads `_run` from the active-baseline prediction blob (or the experiment's blob) тАФ score summary anchor is sourced **from the artifacts that produced the metrics**, not from `project.json` at score time. Closes a latent bug: if user re-points `active_model_id` between extract and score, today's M12.x.d patch mis-attributes.
- Review tabstrip surfaces **every** Run with `_run` as a tab: baseline (`_draft`), pending (`_pending`), and each experiment тАФ uniform shape.

## Hard rules

- **Test data deletable** ([[project_test_data_deletable]]) тАФ pre-prod, old prediction blobs without `_run` are tolerated as "legacy/anonymous" (no chip, no tab); next extract overwrites. No migration script.
- **No new SDK calls** to resolve `_run` at write time тАФ `extract_one` / `experiment_run` / `pre_label` already read `read_active_model` + `read_active_prompt` (or experiment meta). Reuse what's loaded; cost is one dict construction.
- **Run is metadata, not a directory.** No `runs/{run_id}/` filesystem layout in this milestone. Just an envelope on the prediction blob. Defer directory hoist to a later milestone if a real need surfaces.
- **`_run` is informational on prediction blob, not load-bearing for replay.** Predictions remain identified by (slug, filename) for retrieval. `_run.run_id` is a label for UI / chat narration only тАФ not an index key. Keeps SSU; no new pid_index entry, no new `runs/` glob.
- **Reviewed never has `_run`.** Human ground truth has no producing run; `_source` (manual/feedback) is the existing right concept.
- **Score writes the same anchor it reads.** If active baseline `_draft/{f}.json` carries `_run = {extract_model: gemini-2.5-flash, prompt_label: Baseline, тАж}`, the eval summary's `extract_model` / `prompt_label` MUST be those тАФ not whatever `project.json.active_*` happens to be at score time.
- **Existing M12.x.d echo fields stay as the surface contract.** This milestone changes data origin (read from blob), not the field names on `ScoreResultSummary` / `extract_batch` result / `EvalListEntry.meta`. Frontend shape unchanged on those three surfaces.
- **Tool тЖФ HTTP symmetry** ([[feedback_ai_native_api_symmetry]]) тАФ `_run` is data, not a new tool; symmetry invariant test untouched. Only new surface is review-tab baseline rendering (frontend-only, no new tool).

## Spec

### `_run` envelope shape

```json
{
  "entities": [...],
  "_evidence": [...],
  "_run": {
    "run_id": "r_2026-05-21T16-23-04Z_gemini-2.5-flash_pr_baseline",
    "ts": "2026-05-21T16:23:04Z",
    "model_id": "m_default",
    "extract_model": "gemini-2.5-flash",
    "model_label": "Default (gemini-2.5-flash)",
    "prompt_id": "pr_baseline",
    "prompt_label": "Baseline",
    "kind": "baseline"
  }
}
```

`kind` enum: `"baseline" | "experiment" | "pre_label"`. Distinguishes the three write sites for UI rendering (chip color, tab icon).

`run_id` format: `r_{ts}_{extract_model_slug}_{prompt_id}`. Stable per write. Not an index key тАФ it's a label.

### Tabstrip rendering rules

Order:
1. тЬП reviewed (always, editable)
2. baseline (only if `_draft/{f}.json` exists AND has `_run`) тАФ readonly, label = `{extract_model} ┬╖ {prompt_label}` with `Beaker` icon (or similar; distinct from experiment's `FlaskConical`)
3. pending (only if `_pending/{f}.json` exists AND has `_run`) тАФ readonly, label = `{extract_model} ┬╖ {prompt_label}` with `Sparkles` icon, badge `pre-label`
4. each non-archived experiment (existing rendering)

Selection: clicking any tab swaps the right-pane view to that blob's entities (readonly). Annotation tab `'active'` remains the only editable view.

Legacy blobs without `_run`: render no chip, but the underlay-into-form behavior (`stores/review.ts:144`) is preserved тАФ the legacy `_draft` still seeds the editor when reviewed is absent. We just won't show a separate tab.

```python
from typing import Literal, Optional
from pydantic import BaseModel, ConfigDict

RunKind = Literal["baseline", "experiment", "pre_label"]

class RunStamp(BaseModel):
    """Self-identifying envelope on a prediction blob тАФ answers "which
    (model, prompt) produced this?" without a downstream resolver."""
    model_config = ConfigDict(extra="forbid")
    run_id: str
    ts: str
    model_id: Optional[str] = None
    extract_model: Optional[str] = None
    model_label: Optional[str] = None
    prompt_id: Optional[str] = None

    kind: RunKind
```
