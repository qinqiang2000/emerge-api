# M3 — Publish + Prod Fast-Path + API Key Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the `/publish` workflow — when the user types `/publish`, the agent loads the `emerge-publish` skill, runs `readiness_check`, presents the result, asks for explicit confirm, then `freeze_version` (atomic snapshot of `schema.json` + `global_notes.md` + `extract_model` + `extract_params` into `versions/v{n}.json`, `chmod 0o444`, atomic `project.json.active_version_id` update) and `issue_api_key` (random 192-bit `ek_` key, sha256 hashed at rest in `_keys.json`, plaintext returned exactly once via the tool result). Frontend pops a Radix modal that reveals the plaintext on `issue_api_key` results and leaves a hash-only trail card after close. A new prod fast-path router `POST /v1/{pid}/extract` accepts multipart-only file uploads, looks up the API key in an in-memory `KeyStore`, reads `versions/v{active_version_id}.json` (NEVER `schema.json`), and calls `provider.extract` directly — no SDK recursion. Case2 (re-publish v2 with one new field via additive contract diff) is exercised end-to-end via free-form intent in the existing `emerge-extractor` skill — no new slash command.

**Architecture:** Four new tools (`readiness_check`, `contract_diff`, `freeze_version`, `issue_api_key`) live in `app/tools/publish.py` and are registered alongside existing emerge MCP tools. A new `app/security/keys.py` exposes `generate_key`, `sha256_key`, and a singleton `KeyStore` that loads `_keys.json` at startup, reloads on mtime change, and supports `lookup(plaintext) → row | None` and `upsert_for_project(pid, plaintext)` (rotate-by-replace; one key max per project). A new `app/api/routes/publish.py` mounts `/v1/{pid}/extract` (multipart `file=...` + `X-API-Key` header) and `/lab/projects/{pid}/keys/meta` (read-only metadata for the trail card). Bytes from prod-path uploads go straight to `provider.extract` via a new `extract_bytes_with_schema` helper — they NEVER touch the workspace filesystem. The `emerge-publish` SKILL.md is loaded together with `emerge-extractor` whenever the user types `/publish` (extending the same multi-skill loader pattern M2C established for `/improve`). Frontend renders a Radix `ApiKeyRevealModal` triggered by detecting an `issue_api_key` tool result in the chat SSE stream; the plaintext lives only in transient Zustand state and is discarded on modal close.

**Tech Stack:** Backend uses `secrets.token_urlsafe` for key generation, `hashlib.sha256` for hashing, FastAPI's `UploadFile` for multipart, existing `provider.extract` adapter (no SDK recursion), existing `atomic_write_json` + `project_lock` for filesystem invariants, existing `score()` (single-entity, scope-cut per design) for readiness F1. Frontend uses Radix `Dialog` for the modal and a transient Zustand store (`useApiKey`) for the one-time plaintext payload.

## Hard rules (red lines — never violate)

1. **Plaintext API key never lands on disk, in a log, or in a chat JSONL.** It exists exactly twice: as the return value of `issue_api_key` (consumed once by the frontend) and in the response body the user copies. Backend logs use only `key_prefix` (the first 11 chars `ek_xxxxxxxx`) and the short hash suffix.
2. **`_keys.json` stores SHA-256 hashes only.** No salt — random 192-bit keys are not low-entropy passwords; salting adds nothing here.
3. **`POST /v1/{pid}/extract` reads `versions/v{active_version_id}.json` exclusively.** `schema.json` MUST NOT appear in any code path reachable from this endpoint. CI grep enforces this.
4. **`freeze_version` and `issue_api_key` are GATED.** The agent never auto-fires them. The skill instructs the agent to run `readiness_check`, show the result, await explicit user confirmation, and only then call `freeze_version` / `issue_api_key`. The chat composer is the consent surface.
5. **Tools never recurse into the SDK.** `extract_bytes_with_schema` (and every other prod-path code path) calls `provider.extract` directly. No `ClaudeSDKClient` instantiation under `/v1/*`.
6. **Path safety is enforced at the route layer.** `/v1/{pid}/extract` runs `safe_project_id(pid)` BEFORE any file read. The pid in the URL must equal the pid stored in the looked-up key row; mismatch returns `404 not_found` (do not leak whether key or pid was wrong).
7. **`versions/v{n}.json` is immutable.** Atomic write → `chmod 0o444` immediately after replace. Any future code attempting to overwrite is a bug, not a feature.
8. **One key per project.** Re-issuing implicitly invalidates the prior key (rotate-by-replace). The skill warns the user before re-issue if `_keys.json` already has a row for this pid.
9. **Description-as-code remains the only knowledge channel.** `freeze_version` snapshots `schema.json` + `global_notes.md` only. No image few-shot, no example I/O pairs, no counterexamples leak into the frozen version blob.

# backend/app/workspace/paths.py — append at bottom
import re


_VERSION_RE = re.compile(r"^v(\d+)$")


def version_path(workspace: Path, project_id: str, n: int) -> Path:
    return versions_dir(workspace, project_id) / f"v{n}.json"


def parse_version_id(s: str) -> int | None:
    """Return integer n for a version_id like 'v3', or None if malformed.
    n must be >= 1 (we never produce v0)."""
    m = _VERSION_RE.match(s or "")
    if not m:
        return None
    n = int(m.group(1))
    return n if n >= 1 else None


def next_version_n(workspace: Path, project_id: str) -> int:
    """Scan versions/ for v{n}.json files and return max(n)+1, or 1 if empty.
    Ignores _candidate/ subdir (autoresearch outputs) and any non-matching names."""
    vd = versions_dir(workspace, project_id)
    if not vd.exists():
        return 1
    seen: list[int] = []
    for child in vd.iterdir():
        if not child.is_file() or not child.name.endswith(".json"):
            continue
        n = parse_version_id(child.stem)
        if n is not None:
            seen.append(n)
    return max(seen) + 1 if seen else 1
```

# backend/app/security/keys.py
from __future__ import annotations

import hashlib
import secrets

KEY_PREFIX = "ek_"
_KEY_BYTES = 24  # token_urlsafe rounds to 32 chars


def generate_key() -> str:
    """Return a fresh API key in the form `ek_<32 url-safe base64 chars>`.

    24 random bytes -> 192 bits of entropy -> 32 url-safe characters.
    Higher than 128-bit security level, so no salt is needed when hashing —
    rainbow-table attacks against true-random keys are computationally infeasible.
    """
    return KEY_PREFIX + secrets.token_urlsafe(_KEY_BYTES)


def sha256_key(plaintext: str) -> str:
    """Return the lowercase-hex sha256 of the plaintext key. Used for storage and
    constant-time-compare lookup. Plaintext is never persisted."""
    return hashlib.sha256(plaintext.encode("utf-8")).hexdigest()


def key_prefix_display(plaintext: str) -> str:
    """Return the first 11 characters of the key (`ek_` + 8 chars) for safe
    display in logs and the chat trail card. Truncates safely if the input is
    shorter than 11 (only happens in tests)."""
    return plaintext[:11]


def key_hash_short(hash_hex: str) -> str:
    """Return the last 6 hex characters of a sha256 digest, for trail-card display."""
    return hash_hex[-6:]
```

# backend/app/security/keys.py — append at bottom (keep T2 helpers above)
from datetime import datetime, timezone
from pathlib import Path
from typing import Any

from app.workspace.atomic import atomic_write_json


def _iso_now() -> str:
    return datetime.now(timezone.utc).isoformat().replace("+00:00", "Z")


class KeyStore:
    """In-memory cache of `_keys.json` for O(1) prod-path lookups.

    Lookup hashes the plaintext and consults the dict; reload is mtime-based
    (cheap stat call + JSON re-parse). Mutations go through atomic_write_json.

    Discipline: never log, return, or persist plaintext. Only hashes live here.
    """

    def __init__(self, path: Path) -> None:
        self._path = path
        self._by_hash: dict[str, dict[str, Any]] = {}
        self._mtime: float | None = None

    def load(self) -> None:
        self._reload_from_disk()

    def _reload_from_disk(self) -> None:
        if not self._path.exists():
            self._by_hash = {}
            self._mtime = None
            return
        blob = []
        try:
            blob = __import__("json").loads(self._path.read_text())
        except Exception:  # noqa: BLE001 — corrupt key store -> empty cache; never auth-success
            blob = []
        self._by_hash = {row["hash"]: row for row in blob if isinstance(row, dict) and "hash" in row}
        self._mtime = self._path.stat().st_mtime

    def reload_if_changed(self) -> None:
        try:
            mtime = self._path.stat().st_mtime
        except FileNotFoundError:
            if self._by_hash:
                self._by_hash = {}
                self._mtime = None
            return
        if self._mtime is None or mtime != self._mtime:
            self._reload_from_disk()

    def lookup(self, plaintext: str) -> dict[str, Any] | None:
        """Hash the plaintext and look up. Reloads from disk if mtime changed.
        Returns the row dict (with project_id, scope, created_at, last_used) or None."""
        self.reload_if_changed()
        h = sha256_key(plaintext)
        row = self._by_hash.get(h)
        return dict(row) if row is not None else None

    def upsert_for_project(self, project_id: str, plaintext: str, *, scope: str = "extract") -> None:
        """Rotate-by-replace. Removes any existing row for this pid and writes a
        new one with the fresh hash. plaintext is hashed and discarded."""
        self.reload_if_changed()
        # Drop existing rows for this pid
        survivors = [r for r in self._by_hash.values() if r.get("project_id") != project_id]
        new_row = {
            "hash": sha256_key(plaintext),
            "project_id": project_id,
            "scope": scope,
            "created_at": _iso_now(),
            "last_used": None,
        }
        survivors.append(new_row)
        self._path.parent.mkdir(parents=True, exist_ok=True)
        atomic_write_json(self._path, survivors)
        # rebuild index from survivors and refresh mtime
        self._by_hash = {r["hash"]: r for r in survivors}
        self._mtime = self._path.stat().st_mtime

    def update_last_used(self, hash_hex: str, ts: str) -> None:
        """Best-effort persist of last_used. Reloads fresh state, applies the
        update, and atomic-writes. Caller may swallow exceptions to keep the
        prod hot path responsive."""
        self.reload_if_changed()
        rows = list(self._by_hash.values())
        for r in rows:
            if r["hash"] == hash_hex:
                r["last_used"] = ts
                break
        else:
            return
        atomic_write_json(self._path, rows)
        self._by_hash = {r["hash"]: r for r in rows}
        self._mtime = self._path.stat().st_mtime


_STORES: dict[Path, KeyStore] = {}


def get_keystore(workspace: Path) -> KeyStore:
    """Process-wide singleton keyed by workspace path. Loads on first access."""
    from app.workspace.paths import keys_path as _keys_path
    p = _keys_path(workspace)
    if p not in _STORES:
        store = KeyStore(p)
        store.load()
        _STORES[p] = store
    return _STORES[p]

```python
# backend/app/tools/publish.py
from __future__ import annotations

from typing import Any

from app.schemas.schema_field import SchemaField


def contract_diff(
    prev: list[SchemaField],
    candidate: list[SchemaField],
) -> dict[str, Any]:
    """Top-level diff for backward-compat publish gating.

    Breaking changes (return is_breaking=True):
    - removed: a field disappeared (prod clients may rely on it)
    - type_changed: a field's type changed (existing JSON now invalid)
    - enum_narrowed: a field's accepted enum shrunk OR an enum was newly added
      where there was none (prod clients may have sent values now disallowed)

    Safe changes:
    - added: new fields (clients ignore unknown keys)
    - enum_widened: enum grew
    - enum_dropped: free-string accepts any prior value

    Note: depth-0 only. children of array<object> are not deeply diffed in M3.
    """
    prev_by_name = {f.name: f for f in prev}
    cand_by_name = {f.name: f for f in candidate}

    added = sorted(set(cand_by_name) - set(prev_by_name))
    removed = sorted(set(prev_by_name) - set(cand_by_name))

    type_changed: list[dict[str, str]] = []
    enum_narrowed: list[dict[str, Any]] = []
    for name in sorted(set(prev_by_name) & set(cand_by_name)):
        a = prev_by_name[name]
        b = cand_by_name[name]
        if a.type != b.type:
            type_changed.append(
                {"name": name, "prev_type": a.type.value, "candidate_type": b.type.value}
            )
            continue  # don't double-flag enum on a type change
        # enum narrowing
        prev_enum = a.enum
        cand_enum = b.enum
        if prev_enum is None and cand_enum is not None:
            enum_narrowed.append(
                {"name": name, "prev_enum": None, "candidate_enum": list(cand_enum)}
            )
        elif prev_enum is not None and cand_enum is not None:
            if not set(prev_enum).issubset(set(cand_enum)):
                enum_narrowed.append(
                    {
                        "name": name,
                        "prev_enum": list(prev_enum),
                        "candidate_enum": list(cand_enum),
                    }
                )
        # prev_enum present + cand_enum None == widened to free string == safe

    is_breaking = bool(removed or type_changed or enum_narrowed)
    return {
        "added": added,
        "removed": removed,
        "type_changed": type_changed,
        "enum_narrowed": enum_narrowed,
        "is_breaking": is_breaking,
    }

def _last_event_type(jsonl_path: Path) -> str | None:
    """Return the type of the last full JSONL line, ignoring partial trailing line."""
    try:
        text = jsonl_path.read_text(encoding="utf-8")
    except FileNotFoundError:
        return None
    lines = [ln for ln in text.splitlines() if ln.strip()]
    if not lines:
        return None
    # Take the last line that parses as JSON (skip any partial trailing line).
    for ln in reversed(lines):
        try:
            obj = json.loads(ln)
            return obj.get("type")
        except json.JSONDecodeError:
            continue
    return None

        })

            prev_schema = [SchemaField(**f) for f in prev_blob.get("schema", [])]
            diff = contract_diff(prev_schema, schema)
            if diff["is_breaking"]:
                checks.append({
                    "key": "contract_diff_compat",
                    "status": "fail",
                    "detail": (
                        f"breaking changes vs {active_vid}: "
                        f"removed={diff['removed']}, "
                        f"type_changed={diff['type_changed']}, "
                        f"enum_narrowed={diff['enum_narrowed']}"
                    ),
                })
            else:
                detail = f"additive vs {active_vid}: added={diff['added']}" if diff["added"] else f"no changes vs {active_vid}"
                checks.append({"key": "contract_diff_compat", "status": "pass", "detail": detail})
        else:
            checks.append({
                "key": "contract_diff_compat",
                "status": "fail",
                "detail": f"active_version_id={active_vid} but {active_vid}.json missing",
            })
    else:
        checks.append({
            "key": "contract_diff_compat",
            "status": "pass",
            "detail": "no prior active version (first publish)",
        })

    # Soft warnings
    if macro_f1 is not None and threshold <= macro_f1 < 0.85:
        soft.append({
            "key": "f1_borderline",
            "status": "warn",
            "detail": f"macro_f1={macro_f1:.3f} in [{threshold}, 0.85); consider /improve before publish",
        })
    for fs in per_field:
        if fs.support == 0:
            soft.append({
                "key": "field_zero_support",
                "status": "warn",
                "detail": f"field '{fs.field}' has 0 support in reviewed set (untested)",
            })
    md = metrics_dir(workspace, project_id)
    if md.exists() and schema:
        evals = sorted(md.glob("eval_*.json"))
        if evals:
            last = json.loads(evals[-1].read_text())
            if last.get("schema_field_count") != len(schema):
                soft.append({
                    "key": "eval_stale",
                    "status": "warn",
                    "detail": "schema field count changed since last /eval; consider /eval",
                })

    hard_pass = all(c["status"] == "pass" for c in checks)
    return {
        "checks": checks,
        "soft_warnings": soft,
        "hard_pass": hard_pass,
        "macro_f1": macro_f1,
        "n_reviewed": n_reviewed,
    }

# backend/app/tools/publish.py — append below readiness_check
from datetime import datetime, timezone

from app.workspace.atomic import atomic_write_json
from app.workspace.lock import project_lock
from app.workspace.paths import next_version_n


class PublishNotReadyError(Exception):
    """Raised by freeze_version when readiness_check has any hard-fail."""
    def __init__(self, *, error_code: str, error_message_en: str, checks: list[dict] | None = None) -> None:
        self.error_code = error_code
        self.error_message_en = error_message_en
        self.checks = checks or []
        super().__init__(error_message_en)


def _iso_now() -> str:
    return datetime.now(timezone.utc).isoformat().replace("+00:00", "Z")


async def freeze_version(workspace: Path, project_id: str, *, force: bool = False) -> dict[str, str]:
    """Snapshot the current schema + global_notes + model + params into
    versions/v{n}.json, chmod 0o444, and update project.json.active_version_id.

    By default runs readiness_check first and raises PublishNotReadyError if
    any hard gate fails. `force=True` skips readiness — for tests / emergency
    only; do NOT expose this flag as a tool argument.
    """
    if not force:
        readiness = await readiness_check(workspace, project_id)
        if not readiness["hard_pass"]:
            failed = [c for c in readiness["checks"] if c["status"] == "fail"]
            raise PublishNotReadyError(
                error_code="not_ready",
                error_message_en=f"readiness checks failed: {[c['key'] for c in failed]}",
                checks=readiness["checks"],
            )

    async with project_lock(workspace, project_id):
        schema_blob = json.loads(schema_path(workspace, project_id).read_text())
        project_blob = json.loads(project_json_path(workspace, project_id).read_text())
        gn_p = _global_notes_path(workspace, project_id)
        global_notes = gn_p.read_text(encoding="utf-8") if gn_p.exists() else ""

        n = next_version_n(workspace, project_id)
        version_blob = {
            "version_id": f"v{n}",
            "schema": schema_blob,
            "global_notes": global_notes,
            "model_id": project_blob["extract_model"],
            "params": project_blob.get("extract_params") or {},
            "frozen_at": _iso_now(),
        }
        target = version_path(workspace, project_id, n)
        target.parent.mkdir(parents=True, exist_ok=True)
        atomic_write_json(target, version_blob)
        # Lock down the version blob — any future writer must explicitly chmod
        # 0o644 first, which is itself a code-review red flag.
        target.chmod(0o444)

        project_blob["active_version_id"] = f"v{n}"
        atomic_write_json(project_json_path(workspace, project_id), project_blob)

    return {"version_id": f"v{n}"}
```

import logging

from app.security.keys import (
    generate_key,

    key_prefix_display,

async def issue_api_key(workspace: Path, project_id: str) -> dict[str, str]:
    """Mint a fresh API key for this project. Rotates any existing row.

    Returns plaintext exactly once. Caller (chat tool wrapper) ferries it to
    the frontend modal; backend logs only the prefix and short hash.
    """
    plaintext = generate_key()
    h = sha256_key(plaintext)
    store = get_keystore(workspace)
    # Project-level lock guards against issuing twice for the same pid in parallel.
    # Concurrent issues for *different* pids race on _keys.json — acceptable for
    # a single-user lab; documented in scope cuts.
    async with project_lock(workspace, project_id):
        store.upsert_for_project(project_id, plaintext, scope="extract")
    _log.info(
        "issued api key for %s: prefix=%s hash_short=%s",
        project_id,
        key_prefix_display(plaintext),
        key_hash_short(h),
    )
    return {
        "key_plaintext": plaintext,
        "key_hash": h,
        "key_prefix": key_prefix_display(plaintext),
        "created_at": _iso_now(),
    }

_SUPPORTED_EXTS = {"pdf", "png", "jpg", "jpeg"}


def _bytes_to_block(data: bytes, ext: str) -> ContentBlock:
    """Pure helper: bytes + ext (without leading dot) -> a provider ContentBlock.
    Raises ValueError for unsupported extensions."""
    ext = ext.lower().lstrip(".")
    if ext not in _SUPPORTED_EXTS:
        raise ValueError(f"unsupported file extension: {ext!r}")
    b64 = base64.b64encode(data).decode("ascii")
    if ext == "pdf":
        return DocumentBlock(media_type="application/pdf", data_b64=b64)
    media_type = "image/png" if ext == "png" else "image/jpeg"
    return ImageBlock(media_type=media_type, data_b64=b64)

# backend/app/api/routes/publish.py
from __future__ import annotations

import json
import logging
from datetime import datetime, timezone

from fastapi import APIRouter, Depends, File, Header, HTTPException, UploadFile

from app.api.routes._safety import safe_project_id
from app.config import get_settings
from app.provider import get_provider_for_model
from app.schemas.envelope import ErrorEnvelope
from app.schemas.schema_field import SchemaField
from app.security.keys import (
    get_keystore,
    key_hash_short,
    key_prefix_display,
    sha256_key,
)
from app.tools.extract import extract_bytes_with_schema
from app.workspace.paths import (
    parse_version_id,
    project_json_path,
    version_path,
)


router = APIRouter()
_log = logging.getLogger(__name__)

_MAX_UPLOAD_BYTES = 25 * 1024 * 1024  # 25 MB hard cap on prod path


class _AuthContext:
    """Returned from the auth dependency. Carries the pid the key is bound to
    and the key hash (for last_used updates / logging)."""
    __slots__ = ("project_id", "hash_hex", "plaintext_prefix")

    def __init__(self, project_id: str, hash_hex: str, plaintext_prefix: str) -> None:
        self.project_id = project_id
        self.hash_hex = hash_hex
        self.plaintext_prefix = plaintext_prefix


def _envelope(code: str, msg: str) -> dict[str, str]:
    return ErrorEnvelope(error_code=code, error_message_en=msg).model_dump()


async def _resolve_key(x_api_key: str | None) -> _AuthContext:
    if not x_api_key:
        raise HTTPException(status_code=401, detail=_envelope(
            "missing_api_key", "X-API-Key header is required",
        ))
    settings = get_settings()
    store = get_keystore(settings.workspace_root)
    row = store.lookup(x_api_key)
    if row is None:
        raise HTTPException(status_code=401, detail=_envelope(
            "invalid_api_key", "API key is not recognized",
        ))
    return _AuthContext(
        project_id=row["project_id"],
        hash_hex=sha256_key(x_api_key),
        plaintext_prefix=key_prefix_display(x_api_key),
    )


def _iso_now() -> str:
    return datetime.now(timezone.utc).isoformat().replace("+00:00", "Z")


@router.post("/v1/{project_id}/extract")
async def v1_extract(
    project_id: str,
    file: UploadFile = File(...),
    x_api_key: str | None = Header(default=None, alias="X-API-Key"),
) -> dict:
    safe_project_id(project_id)
    auth = await _resolve_key(x_api_key)
    if auth.project_id != project_id:
        # Don't leak whether the key was valid for some OTHER pid.
        raise HTTPException(status_code=404, detail=_envelope(
            "not_found", "no published API at this path",
        ))
    settings = get_settings()
    pj_path = project_json_path(settings.workspace_root, project_id)
    if not pj_path.exists():
        raise HTTPException(status_code=404, detail=_envelope(
            "not_found", "no published API at this path",
        ))
    project = json.loads(pj_path.read_text())
    active_vid = project.get("active_version_id")
    if not active_vid:
        raise HTTPException(status_code=404, detail=_envelope(
            "not_published", "project has no active version; run /publish first",
        ))
    n = parse_version_id(active_vid)
    if n is None:
        raise HTTPException(status_code=500, detail=_envelope(
            "active_version_corrupt",
            f"active_version_id={active_vid!r} is not a valid version id",
        ))
    vp = version_path(settings.workspace_root, project_id, n)
    if not vp.exists():
        raise HTTPException(status_code=500, detail=_envelope(
            "active_version_missing", f"versions/{active_vid}.json is missing",
        ))
    version_blob = json.loads(vp.read_text())
    schema = [SchemaField(**f) for f in version_blob.get("schema", [])]
    if not schema:
        raise HTTPException(status_code=500, detail=_envelope(
            "active_version_corrupt", "frozen schema is empty",
        ))

    # File ext check
    fname = file.filename or ""
    ext = fname.rsplit(".", 1)[-1].lower() if "." in fname else ""
    if ext not in {"pdf", "png", "jpg", "jpeg"}:
        raise HTTPException(status_code=400, detail=_envelope(
            "unsupported_file_type",
            f"file extension {ext!r} not supported; use pdf/png/jpg",
        ))
    content = await file.read()
    if len(content) > _MAX_UPLOAD_BYTES:
        raise HTTPException(status_code=413, detail=_envelope(
            "payload_too_large", f"upload exceeds {_MAX_UPLOAD_BYTES} bytes",
        ))

    provider = get_provider_for_model(version_blob["model_id"])
    try:
        out = await extract_bytes_with_schema(
            content=content,
            filename=fname,
            schema=schema,
            provider=provider,
            model_id=version_blob["model_id"],
            params=version_blob.get("params") or {"temperature": 0.0},
        )
    except Exception as e:  # noqa: BLE001 — bubble through envelope
        _log.warning(
            "v1 extract failure for %s key_prefix=%s hash_short=%s: %s",
            project_id, auth.plaintext_prefix, key_hash_short(auth.hash_hex), e,
        )
        raise HTTPException(status_code=502, detail=_envelope(
            "extract_failed", "extraction failed; see provider logs",
        )) from e

    # Best-effort last_used update; never block response on it.
    try:
        get_keystore(settings.workspace_root).update_last_used(auth.hash_hex, _iso_now())
    except Exception:  # noqa: BLE001
        pass

    return out
```

@router.get("/lab/projects/{project_id}/keys/meta")
async def lab_keys_meta(project_id: str) -> dict:
    """Read-only metadata for the chat trail card. Never returns plaintext.
    If no key has been issued for this pid, returns {key_hash_short: null,
    created_at: null}."""
    safe_project_id(project_id)
    settings = get_settings()
    store = get_keystore(settings.workspace_root)
    store.reload_if_changed()
    row = next((r for r in store._by_hash.values() if r.get("project_id") == project_id), None)
    if row is None:
        return {"project_id": project_id, "key_hash_short": None, "created_at": None, "last_used": None}
    return {
        "project_id": project_id,
        "key_hash_short": key_hash_short(row["hash"]),
        "created_at": row["created_at"],
        "last_used": row.get("last_used"),
    }

<!-- backend/app/skills/emerge_publish.md -->
# emerge-publish (loaded on /publish)

You help the user freeze the current schema as a versioned API and issue an API key.

## Discipline (red lines — never violate)

- `freeze_version` and `issue_api_key` are GATED. ALWAYS run `readiness_check`
  FIRST, present the result in chat, and ASK THE USER to confirm before calling
  `freeze_version`.
- The plaintext API key is shown exactly once via the `issue_api_key` tool
  result. NEVER include the plaintext in your text response — the frontend
  surfaces it via a one-time modal. If the user dismisses the modal without
  saving, they have to re-issue (and the previous key becomes invalid).
- Re-issuing an API key INVALIDATES the prior key. Warn the user before calling
  `issue_api_key` if a key may already exist for this project.
- `freeze_version` writes an immutable `versions/v{n}.json`. The frozen schema
  is what `/v1/{pid}/extract` will serve. Editing `schema.json` afterwards has
  NO effect on prod — only the next `/publish` does.
- Description-as-code: `freeze_version` snapshots `schema.json` + `global_notes.md`
  + the project's `extract_model` / `extract_params`. Make sure these are
  finalized before you publish.
- Backward-compat: contract diff vs the previous active version must be additive.
  Removed / type-changed / enum-narrowed fields are hard fails — surface the
  diff and ask the user to add a new endpoint instead.

## Workflow on /publish

1. Call `readiness_check(project_id)`. Read the returned `{checks, soft_warnings,
   hard_pass, macro_f1, n_reviewed}`.
2. Present the checklist to the user, grouped by status:
   - For each `fail`: explain in plain language what's wrong and the corrective
     action (e.g. "review more docs", "lower `publish_min_macro_f1` via
     `update_project`", "schema has breaking changes vs v1; revert or rename
     instead").
   - For each `warn`: surface plainly. These are advisory.
3. If `hard_pass` is `false`, STOP. Do NOT call `freeze_version`. Ask the user
   to fix and re-run `/publish`.
4. If only soft warnings, ask: "ready to publish v{N}?" Wait for explicit
   confirmation.
5. On confirm, call `freeze_version(project_id)`. Surface the returned
   `version_id`.
6. Ask: "issue a new API key for {pid}?" If a key may already exist (check the
   chat history / a recent `issue_api_key` call), REMIND the user that
   re-issuing invalidates the prior key.
7. On confirm, call `issue_api_key(project_id)`. Do NOT include the plaintext
   in your reply — the frontend will pop a modal. In your reply, simply note
   "API key revealed in modal. Save it now — you cannot view it again."
8. Provide a curl template using the placeholder `<your saved key>`:
   ```
   curl -X POST https://<host>/v1/{pid}/extract \
     -H "X-API-Key: <your saved key>" \
     -F file=@invoice.pdf
   ```

## case2 — re-publish v2 with an added field

If the user is re-publishing after adding a field via free-form intent
("客户反馈缺 BRN 字段") and the existing flow already produced new reviewed
examples + an /eval pass:

- Same `/publish` workflow. `readiness_check` runs `contract_diff` internally
  against the current active version. Provided the only difference is `added`,
  it passes.
- The same API key continues to work; do NOT issue a new key unless the user
  explicitly asks (e.g. they suspect the prior key was leaked).

## Slash commands relevant here

- `/publish` — this skill is loaded alongside emerge-extractor.

## When tools fail

If `freeze_version` raises `not_ready`, surface the failed checks (which the
tool returns in the error envelope) and STOP. There is NO `force` argument.

If `issue_api_key` raises any error, do NOT retry silently. Report the error
to the user and ask them to re-run `/publish` — a failed mint may have left
`_keys.json` partially updated.

## What you do NOT do here

- Do NOT call `extract_one`, `extract_batch`, `derive_schema`, `score`, or
  `start_job` from this skill. Those are extractor-skill territory. If the
  user wants to /eval or /improve before publishing, redirect them to those
  slash commands.
- Do NOT mutate `schema.json` from this skill. If the readiness check says
  "schema is missing X" or "breaking change", route the user back to the
  extractor skill (free-form intent) to make the schema edit.

import json as _json
from app.tools import publish as publish_mod


# Inside build_emerge_mcp, alongside existing @tool wrappers:

@tool(
    "readiness_check",
    "Run the publish readiness checklist for a project. Returns {checks, soft_warnings, hard_pass, macro_f1, n_reviewed}.",
    {"project_id": str},
)
async def t_readiness_check(args: dict[str, Any]) -> dict[str, Any]:
    out = await publish_mod.readiness_check(workspace, args["project_id"])
    return {"content": [{"type": "text", "text": _json.dumps(out)}]}


@tool(
    "contract_diff",
    "Diff the current schema against the active version's frozen schema. Returns {added, removed, type_changed, enum_narrowed, is_breaking}.",
    {"project_id": str},
)
async def t_contract_diff(args: dict[str, Any]) -> dict[str, Any]:
    import json as __j
    from app.workspace.paths import (
        parse_version_id as _pvi,
        project_json_path as _pj,
        schema_path as _sp,
        version_path as _vp,
    )
    pid = args["project_id"]
    schema_blob = __j.loads(_sp(workspace, pid).read_text())
    schema = [SchemaField(**f) for f in schema_blob]
    project = __j.loads(_pj(workspace, pid).read_text())
    avi = project.get("active_version_id")
    if not avi:
        out = {"added": [f.name for f in schema],
               "removed": [], "type_changed": [], "enum_narrowed": [],
               "is_breaking": False, "note": "no prior active version"}
    else:
        n = _pvi(avi)
        prev = []
        if n is not None and _vp(workspace, pid, n).exists():
            prev_blob = __j.loads(_vp(workspace, pid, n).read_text())
            prev = [SchemaField(**f) for f in prev_blob.get("schema", [])]
        out = publish_mod.contract_diff(prev, schema)
    return {"content": [{"type": "text", "text": _json.dumps(out)}]}


@tool(
    "freeze_version",
    "Freeze the current schema as the next versions/v{n}.json. GATED: run readiness_check first and confirm with the user. Returns {version_id}.",
    {"project_id": str},
)
async def t_freeze_version(args: dict[str, Any]) -> dict[str, Any]:
    try:
        out = await publish_mod.freeze_version(workspace, args["project_id"])
        return {"content": [{"type": "text", "text": _json.dumps(out)}]}
    except publish_mod.PublishNotReadyError as e:
        env = {"error_code": e.error_code, "error_message_en": e.error_message_en,
               "checks": e.checks}
        return {"content": [{"type": "text", "text": _json.dumps({"error": env})}]}

// frontend/src/stores/apiKey.ts
import { create } from 'zustand'

import type { RevealPayload } from '../types/publish'


interface State {
  current: RevealPayload | null
  setReveal: (payload: RevealPayload) => void
  clear: () => void
}

/**
 * Transient store for the one-time API-key reveal modal.
 * Plaintext lives ONLY in this in-memory state. We deliberately do NOT
 * persist via zustand/persist or localStorage — the user must save the key
 * before dismissing the modal.
 */
export const useApiKey = create<State>((set) => ({
  current: null,
  setReveal: (payload) => set({ current: payload }),
  clear: () => set({ current: null }),
}))
```

```tsx
// frontend/src/components/Publish/ApiKeyRevealModal.tsx
import * as Dialog from '@radix-ui/react-dialog'
import { Copy, Check, KeyRound } from 'lucide-react'
import { useState } from 'react'

import { useApiKey } from '../../stores/apiKey'


export default function ApiKeyRevealModal() {
  const { current, clear } = useApiKey()
  const [copied, setCopied] = useState(false)
  const [acknowledged, setAcknowledged] = useState(false)

  if (!current) return null

  const handleCopy = async () => {
    try {
      await navigator.clipboard.writeText(current.key_plaintext)
      setCopied(true)
      setTimeout(() => setCopied(false), 1500)
    } catch {
      // surface a softer hint; keep the modal open
      setCopied(false)
    }
  }

  const handleAcknowledge = () => {
    setAcknowledged(true)
    clear()
  }

  return (
    <Dialog.Root open onOpenChange={(o) => { if (!o) clear() }}>
      <Dialog.Portal>
        <Dialog.Overlay className="fixed inset-0 bg-black/40" />
        <Dialog.Content
          className="fixed left-1/2 top-1/2 -translate-x-1/2 -translate-y-1/2 w-[480px] bg-surface text-fg-primary p-6 rounded shadow-lg border border-subtle space-y-4 font-body"
          // Force user action — disable click-outside / Esc to dismiss
          onEscapeKeyDown={(e) => e.preventDefault()}
          onPointerDownOutside={(e) => e.preventDefault()}
        >
          <div className="flex items-center gap-2 text-accent-primary">
            <KeyRound size={18} />
            <Dialog.Title className="font-mono text-sm">API key for {current.project_id}{current.version_id ? ` · ${current.version_id}` : ''}</Dialog.Title>
          </div>
          <Dialog.Description className="text-sm text-fg-secondary">
            这是这把 key 的唯一可见时刻。复制并保存到安全位置 —— 关闭后只剩 hash。
          </Dialog.Description>
          <div className="bg-subtle rounded px-3 py-2 font-mono text-xs break-all select-all">
            {current.key_plaintext}
          </div>
          <div className="flex items-center gap-2">
            <button
              type="button"
              aria-label="copy"
              onClick={handleCopy}
              className="px-3 py-1.5 text-sm bg-accent-info text-white rounded inline-flex items-center gap-1 hover:opacity-90"
            >
              {copied ? <Check size={14} /> : <Copy size={14} />}
              {copied ? '已复制' : '复制'}
            </button>
            <span className="text-xs text-fg-muted ml-auto">created {current.created_at}</span>
          </div>
          <div className="border-t border-subtle pt-3">
            <button
              type="button"
              aria-label="我已保存 — 关闭"
              onClick={handleAcknowledge}
              className="w-full px-3 py-2 text-sm bg-accent-primary text-white rounded hover:opacity-90"
              disabled={acknowledged}
            >
              我已保存到安全位置 — 关闭
            </button>
          </div>
        </Dialog.Content>
      </Dialog.Portal>
    </Dialog.Root>
  )
}
```

import { useProjects } from './projects'

// inside `send`:

if (ev.event === 'tool_result') {
  const d = ev.data as { tool_use_id: string; result_text: string; ok: boolean }
  const projectId = useProjects.getState().selectedId ?? ''
  // Find the active version_id at the moment of issuance — best-effort.
  // We don't have it client-side without a meta fetch; resolve as null and
  // let the modal display "" for version. The agent typically calls
  // freeze_version right before issue_api_key, so the version_id is in the
  // preceding tool_call's result text.
  const versionId = _findRecentVersionId()
  handleToolResult(d, projectId, versionId)

// helpers in module scope:
function _findRecentVersionId(): string | null {
  // Look back through events for a freeze_version tool_call with a parsed result
  const events = useChat.getState().events
  for (let i = events.length - 1; i >= 0; i--) {
    const e = events[i] as any
    if (e.type !== 'tool_call' || e.tool_name !== 'mcp__emerge_tools__freeze_version') continue
    if (typeof e.tool_result === 'string') {
      try { return (JSON.parse(e.tool_result) as { version_id: string }).version_id }
      catch { return null }
    }
    if (e.tool_result && typeof e.tool_result === 'object') {
      return (e.tool_result as { version_id?: string }).version_id ?? null
    }
    break
  }
  return null
}

function handleToolResult(
  d: { tool_use_id: string; result_text: string; ok: boolean },
  projectId: string,
  versionId: string | null,
) {
  // Find the originating tool_call so we know which tool emitted this result.
  const events = useChat.getState().events
  const parent = events.find(e => e.type === 'tool_call' && e.tool_use_id === d.tool_use_id) as any

  let resultPayload: unknown = d.result_text
  if (parent && parent.tool_name === 'mcp__emerge_tools__issue_api_key') {
    try {
      const parsed = JSON.parse(d.result_text) as {
        key_plaintext?: string; key_hash?: string; key_prefix?: string; created_at?: string
        error?: { error_code: string; error_message_en: string }
      }
      if (parsed.error) {
        // Surface as an error event; do not pop modal
        useChat.setState(s => ({
          events: [...s.events, { type: 'error',
            error_code: parsed.error!.error_code,
            error_message_en: parsed.error!.error_message_en }],
        }))
        // Result text becomes a generic redacted blob
        resultPayload = { redacted: true, error: parsed.error.error_code }
      } else if (parsed.key_plaintext && parsed.key_hash && parsed.key_prefix && parsed.created_at) {
        // Pop the reveal modal
        useApiKey.getState().setReveal({
          key_plaintext: parsed.key_plaintext,
          key_hash: parsed.key_hash,
          key_prefix: parsed.key_prefix,
          created_at: parsed.created_at,
          project_id: projectId,
          version_id: versionId,
        })
        // Redact the chat event
        resultPayload = {
          redacted: true,
          key_prefix: parsed.key_prefix,
          key_hash_short: parsed.key_hash.slice(-6),
          created_at: parsed.created_at,
        }
      }
    } catch {
      // Keep raw text if parsing failed
    }
  }

  useChat.setState(s => ({
    events: s.events.map(e => {
      if (e.type === 'tool_call' && e.tool_use_id === d.tool_use_id) {
        return { ...e, tool_result: resultPayload, ok: d.ok }
      }
      return e
    }),
  }))
}

## Self-Review notes

- **Multi-entity** stays scope-cut (open follow-up #1). M3 readiness inherits the `entities[0]` limitation only in the F1 calculation; the prod path itself returns whatever the model emits (full `entities` array). If a project has multi-entity PDFs, prod still works; readiness's macro_f1 may be misleadingly low because `score()` only graded the first entity.
- **Concurrent issue_api_key for different pids** races on `_keys.json` (project_lock is per-pid, not workspace-wide). Single-user lab — accepted. A workspace-wide flock would fix this; deferred.
- **`_evidence` round-trip on prod path.** `extract_bytes_with_schema` returns whatever the provider emits — `_evidence` is preserved if the model emitted it. The system prompt continues to ask for it. No new test coverage in M3 for evidence on prod path; M2C unit tests for `extract_one_with_schema` already cover the model_dump shape.
- **Audit log for `/v1`.** `last_used` is updated on success; per-call audit JSONL is deferred. Open follow-up.
- **`KeyStore._by_hash` accessed directly in `keys/meta`** route (T10 step3). Slight boundary violation; if a future task adds a public `rows()` method, replace the comprehension. Not blocking.
- **Frontend modal is force-modal** (Esc + click-outside disabled). The user MUST click "我已保存" to dismiss. This is intentional — the plaintext is gone forever after dismissal. If accessibility audits flag this, add a focus trap warning rather than re-enabling click-outside.
- **`_select_system_prompt`** continues to be a string-prefix matcher rather than a full intent classifier. With `/improve` and `/publish` it's two cases; `/feedback` and others stay routed through the default extractor skill via free-form text. If the count grows past 4-5, refactor to a registry.
- **The `freeze_version` tool wrapper** does not expose `force=True`. The Python function does (for tests + emergency). Keep this asymmetry: agent-callable surface stays gated.
- **Schema validation in T13 contract_diff tool wrapper** uses `SchemaField(**f)` over the raw blob. If a frozen v{n}.json has a field shape the current `SchemaField` model rejects (model migration), the wrapper raises. This is a feature, not a bug — surfaces the schema-version mismatch loudly.
